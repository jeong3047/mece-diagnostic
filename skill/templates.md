# Templates — MECE Diagnostic 템플릿

## Issue Tree 템플릿

```
## Issue Tree: [최상위 질문]

├── [이슈 A] — [한 줄 설명]
│   ├── [하위 A-1]
│   └── [하위 A-2]
├── [이슈 B] — [한 줄 설명]
│   ├── [하위 B-1]
│   └── [하위 B-2]
└── [이슈 C] — [한 줄 설명]
    ├── [하위 C-1]
    └── [하위 C-2]

분해 전략: [수학적 / 프로세스 / 세그먼트 / 내부-외부]
MECE 검증: [상호배타성 확인] + [포괄성 확인]
```

## Why Tree 템플릿

```
## Why Tree: 왜 [문제]가 발생하는가?

├── [원인 그룹 1]
│   ├── [세부 원인 1-1] — 근거: [...]
│   └── [세부 원인 1-2] — 근거: [...]
└── [원인 그룹 2]
    ├── [세부 원인 2-1] — 근거: [...]
    └── [세부 원인 2-2] — 근거: [...]

분해 전략: [사용한 전략]
✅ MECE 검증:
  - 상호배타: [그룹 간 겹침이 없는 이유]
  - 포괄적: [이 분해가 전체 원인을 커버하는 이유]
```

## 가설 카드 템플릿

```
### 가설 #[N]: [가설 제목]

- 가설: "[원인 X] 때문에 [문제 Y]가 발생한다"
- 근거: [왜 이 가설이 유력한지 — 로그, 코드, 타이밍 등]
- 반증 조건: [이것이 확인되면 가설 기각]
- 검증 도구: [사용할 도구 — Grep/Read/git/Bash/WebSearch/MCP 등]
- 검증 데이터: [구체적으로 어떤 파일/로그/메트릭을 확인할지]
- 반증 신호: [어떤 결과가 나오면 이 가설을 기각하는지]
- 예상 소요: [검증에 걸리는 시간/노력]
```

## Priority Matrix 템플릿

```
## Priority Matrix

| # | 가설 | Impact | Evidence | 우선순위 |
|---|------|--------|----------|----------|
| 1 | [가설 제목] | H/M/L | H/M/L | [I×E 순위] |
| 2 | [가설 제목] | H/M/L | H/M/L | [I×E 순위] |
| 3 | [가설 제목] | H/M/L | H/M/L | [I×E 순위] |

Impact 기준: 영향 받는 범위 (사용자 수, 파일 수, 시스템 수)
Evidence 기준: 검증 용이성 (로그 존재, 재현 가능, 코드 추적 용이)

→ 검증 순서: #[N] → #[N] → #[N]
```

## Pruning Record 템플릿

```
## Pruning Record

| # | 가설 | 판정 | 근거 | 비고 |
|---|------|------|------|------|
| 1 | [가설] | ✅ 확정 / ❌ 기각 / ⏸ 보류 | [판정 근거] | [추가 메모] |
| 2 | [가설] | ❌ 기각 | [반증 데이터] | — |

MECE 보존 확인: Pruning 후 남은 가지들이 여전히 MECE한가? [예/아니오 + 설명]
```

```
**확정 근거:**
1. [검증 데이터 1]

3. [기각된 대안 가설 요약]

**영향 범위:** [어디까지 영향을 미치는지]
```

## What Tree 템플릿 (Deep 전용)

```
## What Tree: [문제]를 어떻게 해결할 것인가?

├── 단기 (즉시 적용 가능)
│   ├── [방안 1] — 효과: [...] / 비용: [...] / 리스크: [...]
│   └── [방안 2] — 효과: [...] / 비용: [...] / 리스크: [...]
└── 장기 (구조적 개선)
    ├── [방안 3] — 효과: [...] / 비용: [...] / 리스크: [...]
    └── [방안 4] — 효과: [...] / 비용: [...] / 리스크: [...]

✅ MECE 검증: [단기/장기 분해의 MECE 확인]

→ 권고: [최종 추천 방안과 이유]
```

## SW 컨텍스트 데이터 수집 체크리스트

분석 시 확인할 수 있는 데이터 소스:

```
- [ ] 에러 메시지 / 스택 트레이스
- [ ] 관련 로그 (서버, 클라이언트, 빌드)
- [ ] git log / git blame (최근 변경 이력)
- [ ] git diff (변경 내용)
- [ ] 테스트 실행 결과
- [ ] 네트워크 요청/응답 (API 호출)
- [ ] 환경 설정 (env, config)
- [ ] 의존성 버전 (package.json, lock file)
- [ ] CI/CD 파이프라인 로그
- [ ] 모니터링/메트릭 데이터
- [ ] 관련 문서 (CLAUDE.md, README, API docs)
- [ ] 유사 이슈 (GitHub issues, PR comments)
```

## Quality Checklist

분석 완료 후 최종 확인:

```
- [ ] 모든 분해가 MECE한가? (겹침/누락 없음)
- [ ] 가설을 데이터 전에 세웠는가?
- [ ] 기각된 가설의 Pruning 기록이 있는가?
- [ ] Pruning 후 남은 가지들이 MECE를 유지하는가?
- [ ] 근본 원인에서 해결 방안으로의 논리적 연결이 있는가?
- [ ] 복잡도 레벨이 적절했는가? (격상이 필요했는데 안 했는가?)
- [ ] 분해 전략 선택이 문제에 적합했는가?
```
